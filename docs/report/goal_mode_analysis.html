📋 文件概览

🎯 核心定位

Goal Mode 是 generic-agent 的自主持续工作模式，让 Agent 能够像人类一样自主规划、持续推进任务，直到时间预算耗尽。与传统的一次性任务执行不同，Goal Mode 强调的是"找事做"和"持续改进"。

⏱️ 时间预算驱动

基于预算时间自动控制任务生命周期，预算耗尽自动收口

🔄 轮次控制

max_turns 防空转上限，避免Agent空转浪费资源

📁 状态持久化

JSON文件存储状态，支持进程重启后恢复

🎯 多Worker协作

Goal Hive模式支持Master调度多个Worker并行工作

🎯 Goal Mode 核心实现

状态管理机制

goal_state.json 结构: { "objective": "用户原话目标", "budget_seconds": 10800, // 最少3小时 "start_time": , // 启动时间戳 "turns_used": 0, // 已用轮次 "max_turns": 200, // 防空转上限 "status": "running", // running | wrapping_up | done_budget "done_prompt": "" // 收口提示(可选) }

check() 函数 — 核心决策逻辑

1

加载状态

从 goal_state.json 读取当前状态，无文件返回 /exit

2

状态检查

status != 'running' 时直接返回 /exit

3

预算/轮次检查

remaining <= 0 或 turn > max_turns → 进入收口阶段

4

生成Prompt

根据状态返回 CONTINUATION_PROMPT 或 BUDGET_LIMIT_PROMPT

on_done() — 收口处理

def on_done(result): state = _load() if state is None: return if state.get('status') == 'wrapping_up': state['status'] = 'done_budget' state['end_time'] = time.time() _save(state)

Prompt 模板设计

模板 触发条件 核心指令
CONTINUATION_PROMPT 正常运行时 禁止停止、持续推进、主动找改进点
BUDGET_LIMIT_PROMPT 预算耗尽 总结进展、列出未完成、清理临时文件

🐝 Goal Hive 协作模式

💡 核心概念

Goal Hive = Goal Mode 的多 Worker 协作协议，通过中央公告板(BBS)实现任务分发和结果回收。Hive模式单独运行，不要和plan/supervisor/subagent混杂。

启动流程

1

准备 BBS

选择空闲端口 PORT 和协作 key BOARD_KEY，创建数据目录

2

启动 BBS 服务

后台启动 agent_bbs.py 服务进程

3

发布任务公告

在BBS发帖，包含目标、Master职责、Worker指引

4

启动 Worker

后台启动首个 worker 进程

5

启动 Hive Master

按 goal_mode_sop.md 后台启动 Master，分配时间预算

Hive Master 职责

  • 负责任务调度和团队组织，不允许亲自干活导致 Worker 空转
  • 终极目标是要做到完美的找不到任何问题的任务交付结果
  • 针对任务目标设计要做的子任务，发到 BBS 上
  • 如果子任务很多，可以拉起更多 Worker
  • 只要时间没到，就持续验收结果、检查问题、寻找下一个改进点
  • 时间没到不允许交付，必须头脑风暴找改进点
  • 收口时关闭所有 Worker，发帖宣告任务结束

⚙️ 配置说明

Goal State 必需字段

字段 类型 说明 建议值
objective string 用户原话目标 必填
budget_seconds int 时间预算(秒) ≥10800 (3小时)
start_time float 启动时间戳 time.time()
max_turns int 最大轮次上限 200
status string 运行状态 "running"

📝 总结

🎯 Goal Mode 设计亮点

  • 预算驱动：时间预算作为核心约束，自动控制生命周期
  • 状态持久化：JSON文件存储，支持进程重启恢复
  • 防空转机制：max_turns 防止Agent空转
  • 多Worker协作：通过BBS实现任务分发和结果回收
  • 角色分离：Master负责调度，Worker负责执行

适用场景

  • 用户给出开放目标 + 时间预算
  • 需要持续优化、迭代改进的任务
  • 需要多Agent并行处理的大型任务
  • "没事也找事干"的持续改进场景

关键约束

  • 预算最少3小时，避免蜻蜓点水
  • 禁止提前交付，必须持续推进
  • 不更新全局记忆，在工作文件夹记录进度
  • Hive模式单独运行，不混杂其他模式
