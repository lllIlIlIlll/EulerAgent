⚙️ 8 条操作规则

Rule 1 · Search First

文件名搜索必须用 es；Web 搜索必须用 Google；先检查 cwd，不猜测路径。

Rule 2 · Cross-verify

不信任摘要，必须在详情页核实数字。

Rule 3 · Encoding Safety

用 file_read 不用 cat/type；修改前必读；直接 import memory 模块。

Rule 4 · Close the Loop

物理模拟后确认结果；3次失败请求干预；完成 Git 工作流。

Rule 5 · Processes

不无条件杀 Python（会杀自己）；用精确 PID；不用 os.kill 做存活检查。

Rule 6 · Windows

Windows 下优先用 win32gui 枚举窗口标题获取 GUI 状态。

Rule 7 · Web JS

用原生 setter + event chain 填表单；点击前检查 disabled；扫描为空则等待重扫。

Rule 8 · SOP

读 SOP 不要靠记忆；复杂任务/规划场景读 plan_sop。

💡 关键洞察

洞察 1

L1 是索引而非存储。它只是指针，真正的事实数据在 L2。Agent 通过 L1 快速定位所需资源，无需遍历全部记忆文件。

洞察 2

用户偏好在本次会话后从 Working 同步到 L2，实现了持久化。下次新会话启动时，Agent 可直接从 L2 读取偏好，无需重新设置。

洞察 3

5 层架构中，L0（META-SOP）是宪法层，L3 是最大的工具层（14 SOPs + 8 工具），L4 是历史归档。各层职责单一，边界清晰。

洞察 4

15 个工具分布在 5 大能力域，Browser 域能力最丰富（7 项高级能力）。这说明 Agent 的核心能力是 Web 自动化。

洞察 5

8 条 RULES 覆盖了搜索/验证/编码/进程/跨平台/Web JS/规划等关键场景，是 Agent 行为的操作手册，降低试错成本。
