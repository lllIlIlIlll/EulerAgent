规则 4: Let it crash / Failure radius

错误应该明确暴露，便于快速定位。禁止使用可被禁用的 assert 进行运行时校验。

R4-01  不通过 llmcore.py:679

问题：使用 assert 进行运行时参数校验。Python 在 -O 模式下会跳过所有 assert 语句，导致校验静默失效。

def ask(self, msg):
    assert type(msg) is dict    # ← 应改为显式 raise TypeError

上下文：该方法是 ClaudeSession.ask() 和 OAISession.ask()（tool_use 块处理链），msg 必须为 dict。若 assert 被跳过，后续 self.history.append(msg) 会静默存入错误类型，难以排查。

修复：

if not isinstance(msg, dict): raise TypeError(f"Expected dict, got {type(msg).__name__}")
R4-02  不通过 llmcore.py:909

问题：同样使用 assert 校验 MixinSession 初始化参数。

groups = {is_native(s) for s in self._sessions}
assert len(groups) == 1, f"MixinSession: sessions must be in same group (Native or non-Native), got {[type(s).__name__ for s in self._sessions]}"

修复：

if len(groups) != 1:
    raise ValueError(f"MixinSession: sessions must be in same group, got {[type(s).__name__ for s in self._sessions]}")
